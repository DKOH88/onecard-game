# 버전 업데이트 워크플로우

버전 업데이트 시 아래 단계를 **반드시** 따릅니다:

## 1. 코드 수정
- `online.html`에서 `version-badge` div 내용을 새 버전으로 변경
- 예: `<div class="version-badge">v2.5.8</div>`

## 2. Git 커밋 및 푸시
```powershell
git add online.html
git commit -m "v{버전}: {변경 내용 요약}"
git push
```

## 3. 로그 파일 업데이트 (필수!)
**반드시** `log/version_history.txt` 파일에 새 버전 추가:
- 버전 번호
- 업데이트 시간 (HH시MM분 형식)
- 변경 내용 요약

예시:
```
v2.5.8     | 09시51분     | 새로운 기능 추가
```

## 버전 번호 규칙
- 형식: `vX.Y.Z` (각 자리는 0~9만 사용)
- v2.5.9 다음은 v2.6.0
- v2.9.9 다음은 v3.0.0

## ⚠️ 주의사항
- 로그 파일 업데이트를 **절대** 빠뜨리지 말 것!
- 시간은 현재 시간 기준으로 기록
